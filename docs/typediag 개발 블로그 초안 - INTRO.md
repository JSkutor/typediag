# Intro

> TypeDiag는 CPM을 넘어 3D 지연 지형과 다양한 통계 정보로 타건을 진단하는 한글 특화 타자 연습 서비스다.

나는 오타 습관을 고치고 싶었을 뿐이다.
**TypeDiag** 토이 프로젝트로 시작해, 여러 아이디어가 달라붙어 테스트 포함 2.7만 줄 규모가 된 웹앱이다.
이 시리즈에서는 지난 2주간의 개발 과정과 아키텍처를 공유하려 한다.

이 프로젝트는 거의 100% **에이전트 코딩**으로 이루어졌다.
Antigravity 2.0 과 Cursor 를 주로 사용했다.

---
## 개발 동기
타자연습을 해야겠다는 동기가 먼저 있었다.
처음에 습관을 잘못 들여, 오타를 너무 많이 내는 것이 문제였다.
한 곳에서 5번을 틀리고 **깊은 빡침을 느끼며 백스페이스를 연타한 경험**을 나만 갖고 있지는 않을 거다.

todolist에 적힌 타자연습을 보고 직접 만들어 쓰겠다고 생각한 이유는 크게 두 가지다.

**1. '한글' 타자 연습 프로그램의 부재.**
한컴타자연습과 Monkeytype 이라는 거대한 서비스들이 있지만 그 중간 지점을 봤던 것 같다.
**2. 지문 내용에 대한 불만족.**
무엇보다 LLM의 시대에 문학 작품 단편이나 출처도 모르는 글을 의미없이 따라치고 싶지 않았다.

타이밍도 잘 맞았다.
방학이 아니었다면 이런 결과물까지는 안 나왔을 거다.

## 기능 구상
코드 작업 시작 전에 계획하고 있던 기능은 많지 않았다.
1. qwerty 키보드 2차원 키 배열 위에 latency를 z로 하는 **mesh**를 그리면 보기 편하겠다.
2. 한글 자모 조합 과정에서의 **오타 판별**을 정확하게 해야겠다.
3. 문장(target text)을 **언어모델** 써서 만들어야겠다.
이 정도뿐이었다.

mesh 시각화 부분을 메인 기능으로 구상 중이었다.
전처리와 smoothing, 그리고 mesh화가 핵심 요소여서 마냥 글이나 머리로만 생각하는 것은 한계가 있었다.
그래서 웹 환경 등을 생각 안 하고 전처리 결과, 시각화를 즉각적으로 피드백 할 수 있게 **python 기반 서브 프로젝트**를 시작했다.
raw 키 입력 데이터부터, 전처리, 시각화 전반의 파이프라인을 **SKDM (Spatial Keystroke Dynamics Model)**이라고 이름지었다.

## 서브 프로젝트
이후 시리즈에서 소개할 구체적인 SKDM의 로직은 생략한다.
서브 프로젝트에서 기본 로직의 기반은 완성된 상태였다.
그리고 나서 mesh의 시각적인 부분을 다듬는 중 문득 '내가 여기서 뭐 하고 있는 건가' 생각이 들었다.
어차피 메인 프로젝트로 옮기면 디테일을 다시 처음부터 잡아야만 할 것 같았다.
그래서 서브 프로젝트는 애매한 상태로 남겨놓고 본격적으로 메인 프로젝트를 시작했다.
서브 프로젝트 작업은 구현 가능성 정도만 확인한 수준에 그쳤다.

문제는 생각보다 빨리 생겼다.
reference가 될 만한 코드가 적었던 것이 화근이었던 것 같다.
> 아무리 쪼개서 작업하려 해도 에이전트로 구현이 안 됐다.

이미 구상을 끝낸 아이디어가 에이전트로 구현이 안 된다는 게 애초에 신선한 경험이었는데,
심지어 이미 python 프로젝트에서는 구현이 되어있다는 사실에 더 답답했다.
몇 시간 더 했으면 됐을 수도 있었겠지만 인내심이 거기까지는 가지 못했다.

다시 서브 프로젝트로 돌아가서 모델을 python 기반으로 완벽히 다듬고,
하나의 python 파일을 그대로 가져와서 이식하는 방향으로 계획을 틀었다.

결과는 성공적이었다.
계속 엇나가던 에이전트가, 코드를 쥐어주니까 마치 내가 멍청하게 말했었다는 듯이 **parity test** 까지 완벽히 구현해냈다.

이후 대략적인 과정은 다음과 같다.
세부 로직과 과정들은 이후 글에서 다룰 것이다.
- Next.js App Router 기반 UI 스켈레톤 구성,
- 로직 테스트를 위한 세션, DB, 로컬 저장 구현,
- 입력 중 실시간 오타 판별을 위한 자체 휴리스틱 알고리즘(MVSA) 설계,
- 다양한 타건 진단(diagnostics) 지표 추출,
- 연습 모드 다변화,
- 로컬에서 PostgreSQL로 DB 마이그레이션

## 시리즈 목차
개발 초기 부분과 개괄적인 소개는 여기서 마친다.
SKDM과 MVSA 말고도 밀도 높은 기능들이 많이 남아있다.
시맨틱 벡터, 언어모델, 분절회귀 등 다양한 분야의 개념들을 활용해 기능들을 구현했다.
어쩌다 보니 그냥 내가 아는 걸 다 가져다 쓴 것 같은 느낌도 든다.
하지만 이 프로젝트의 핵심은 모든 기술이 **필요에 의해 개발**되었다는 것이다.

MVP까지의 프로젝트를 소개하기 위해 다음 6편의 글을 계획 중이다.

**1. 한글 입력 도중에서의 오타 판별
**	- 한글 자소 조합 휴리스틱 상태머신 (MVSA)
**2. 병목 구간의 3D 시각화
**	- Delaunay 삼각분할, Graph Laplacian Smoothing (SKDM)
**3. 개별 키들의 미시적 시각화
**	- Cylindrical Coordinate 매핑
**4. 다양한 타건 진단
**	- accumulator 기반 분절회귀, 구름타법, NGram
**5. 개인 문장 생성의 비용 최적화
**	- pgvector 기반 시맨틱 캐싱
**6. 약한 입력 훈련
**	- MLP LM 의 학습, 로짓 반전과 추론


앞으로의 글에서 타자연습 서비스 개발에서 이런 단어들이 나오게 된 이유를 하나하나 풀어보려 한다.
다음 글에서는 직접 설계한 한글 특화 오타 판별 알고리즘을 소개하겠다.
