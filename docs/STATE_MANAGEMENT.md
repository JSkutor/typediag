# State Management & Session Lifecycle

This document provides a comprehensive overview of TypeDiag's state management architecture, focusing on Zustand slices, global stores, and the core session lifecycle business rules.

## 1. Store Architecture Overview

TypeDiag separates state concerns across multiple Zustand stores to maintain a clean, performant, and scalable architecture.

### Global & UI Stores

- **`useAppStore` (`src/store/useAppStore.ts`)**: Manages global user preferences.
  - **State**: `language` (`ko` | `en`), `layout` (`qwerty` | `dvorak`), `showLiveHeatmap`.
  - **Persistence**: Persisted to `localStorage` under `typediag-settings` to ensure preferences remain across sessions.

- **`useWorkspaceStore` (`src/store/useWorkspaceStore.ts`)**: Manages dashboard UI and diagnostics visualization state.
  - **State**: `uiState` (`practice` | `measuring` | `diagnostics`), `diagnosticMode` (`surface` | `cylindrical` | `backspace` | `finger`), `focusedKey`, `dynamicScale`.
  - **Data**: Holds data required for 3D visualization like `keyStats`, `triangles`, and `analysisEvents`.

- **`useFeedbackStore` (`src/store/useFeedbackStore.ts`)**: Manages feedback submission flow.
  - **State**: `submitStatus` (`idle` | `submitting` | `success` | `error`), `submitError`.
  - **Logic**: Submits freeform typing messages to `feedbackServiceClient` and handles UX transitions back to normal mode upon success.

### Typing Store (`useTypingStore.ts`)

The primary store for high-frequency typing state. It uses the Zustand slice pattern to merge multiple domains into a single reactive store.

```text
src/store/
├── typingSlices/
│   ├── createInputSlice.ts      # Typing buffers, target text, modes
│   ├── createKeystrokeSlice.ts  # Key press/release tracking
│   ├── createSessionSlice.ts    # Run/page lifecycle & data persistence
│   ├── createTopicSlice.ts      # Topic mode (generation, navigation)
│   ├── createNormalModeSlice.ts # Normal mode specific behaviors
│   ├── types.ts                 # Store types
│   └── utils.ts
└── useTypingStore.ts            # Root store assembly
```

## 2. Typing Slices Breakdown

### 2.1. Input Slice (`createInputSlice.ts`)

Manages text targets, user inputs, and different typing modes.

- **Modes**:
  - `normal`: Standard practice mode.
  - `topic`: Practice using sentences generated by AI based on a specific topic.
  - `hardcore`: Challenging, random length text generation.
  - `feedback`: Uses the typing interface to let the user type and submit feedback.
- **Key State**:
  - `targetText`, `targetId`, `targetLanguage`: Current active sentence details.
  - `typedText`: Display text (e.g., assembled Hangul).
  - `qwertyBuffer`: Raw physical keystrokes (e.g., `gks`).
  - `alignments`: Results from the MVSA algorithm mapping the typed sequence against the target for precise diff highlighting.
- **Delegation**: Depending on the `mode`, actions like `nextTarget` or physical key presses are delegated to mode-specific handlers (like `createTopicSlice` or `createNormalModeSlice`).

### 2.2. Topic Slice (`createTopicSlice.ts`)

Handles the lifecycle of Topic Mode sentences, including fetching from the database and falling back to LLM generation.

- **State**: `isTopicInputActive`, `isTopicLoading`, `isTopicGenerating`, `topicGenerateError`.
- **Logic**:
  - `fetchTopicTarget`: Attempts to fetch sentences from `POST /api/practice/topic`. If none exist (404), it falls back to OpenAI LLM generation (`fetchTopicGenerateWithRetry`).
  - **Prefetching**: Automatically pre-fetches more sentences if the pool drops below a threshold.

### 2.3. Keystroke Slice (`createKeystrokeSlice.ts`)

Calculates physical key latencies and hold durations.

- **State**: `events` (array of `KeyEvent`s), `lastKey`, `lastKeyAt`, `pressedKeys`.
- **Logic**:
  - `recordKey`: Appends key down events. At this stage, `holdDurationMs` is null.
  - `handlePhysicalKeyRelease`: Finds the associated keydown event and computes `holdDurationMs`. Handles edge cases like repeated OS keystrokes (e.g., holding backspace).

### 2.4. Session Slice (`createSessionSlice.ts`)

Coordinates when to start a "run", when to end a page, and how to persist data securely via `sessionServiceClient`.

- **State**: `status` (`idle`, `running`, `done`), `currentRunId`, `pageMetricsFlash`.
- **Logic**:
  - Uses a **Pending Page Save** (`pendingPageSave.ts`) mechanism. Before transitioning to a new sentence (`saveCurrentPageIfDone`), the current page is captured in a pending state and flushed asynchronously.
  - On page finish, fires PostHog analytics (`typing_page_completed`) with computed WPM, CPM, and accuracy.

## 3. Backspace Deletion Logic

Unlike standard `<input>` fields, TypeDiag mutates the `qwertyBuffer` directly and re-assembles the Hangul.

- **English**: Simple character slice off the end.
- **Korean**:
  - Removes the last **complete visual character** if deleting past previously confirmed text (`maxTypedTextLength`).
  - Otherwise, removes single jamo characters.
- Every backspace is treated as `isCorrect: true` and generates a dedicated `KeyEvent` for later analysis.

## 4. Diagnostics & 3D Analytics

When the `diagnosticMode` is set to `cylindrical`, the state relies on dedicated hooks rather than Zustand to process complex aggregations without blocking the main UI thread.

- **Hook**: `useCylindricalDiagnostics.ts` computes the diagnostics transition stats based on `workspaceStore.analysisEvents`.
- **Logic**: Uses a 1-pass accumulator (`buildDiagnosticsAccumulator`) to process incoming keystrokes into reference and outgoing transitions based on the `focusedKey`.

---

## 5. Session Lifecycle Rules

`SessionService` manages isolated run tracking.

1. **3-Minute Inactivity Rule**: If the gap since the last active page or run start is greater than 3 minutes, the run finalizes, and a new `runId` is generated.
2. **5-Minute Gap Splitting Rule**: If a single page takes more than 5 minutes due to inactivity mid-sentence, the completed page is retroactively split into a new run.
3. **Outlier Latency Correction**: Keypress latencies exceeding 3 seconds are clamped to the sentence's average to prevent ruining the WPM/CPM math. Gap durations are mathematically excised from `startedAt` timestamps.
